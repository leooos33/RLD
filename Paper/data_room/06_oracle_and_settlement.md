# Oracle Architecture and Settlement

## Abstract
The integrity of Rate-Level Derivatives and Parametric Credit Default Swaps relies exclusively on the precision, resilience, and manipulation-resistance of the underlying interest rate data feed. The Oracle and Settlement architecture introduces a multi-layered, trustless infrastructure utilizing Zero-Knowledge State Attestations and Symbiotic Restaking to withstand extreme volatility, flash loans, and coordinated governance attacks.

## 1. The Time-Weighted Average Rate (TWAR)

DeFi interest rates are highly responsive to ephemeral utilization shocks or atomic flash-loan manipulations. Settling derivatives on instantaneous spot rates introduces unacceptable liquidation risk.

To mitigate this, the Oracle calculates a Time-Weighted Average Rate (TWAR) over a specified, configurable rolling window (e.g., 30-60 minutes).

$$ P_{index}(t) = K \cdot \left( \frac{\sum_{i=1}^{N} r_i \cdot \Delta t_i}{\sum \Delta t_i} \right) $$

This mathematical smoothing function acts as a low-pass filter. An atomic flash loan driving utilization to 100% for a single block will have a mathematically negligible impact on the TWAR, effectively ignoring noise while ensuring that sustained, genuine liquidity crises correctly pull the index price toward the settlement threshold.

## 2. Zero-Knowledge State Attestation

The fundamental vulnerability of traditional oracle networks is that they rely on operators to *report* subjective data, creating a massive attack vector requiring 66% consensus to compromise. The RLD infrastructure completely bypasses data reporting by transitioning to Zero-Knowledge State Attestation.

Rather than trusting an oracle to report the interest rate, the system requires operators to generate and submit a Zero-Knowledge Proof (ZKP) verifying the exact historical state of the underlying lending protocol.

$$ \pi_t = \text{ZKProof}\Big(\text{source}_{chain}(t) \to \text{value}_t\Big) $$

**Attack Surface Reduction:**
Because the smart contract verifies the ZKP natively on-chain, the operator mathematically cannot fabricate data. A valid proof guarantees the data is the absolute truth of the blockchain's state history. This reduces the attack surface from an active *correctness attack* (lying about data) to a passive *liveness attack* (refusing to submit proofs).

### 2.1 Liveness-as-Default (Fail-Safe)
If operators collude to halt proof submission during a crisis, the smart contract utilizes a fail-safe mechanism: silence defaults to zero. 

If no valid proof is submitted within a time window $T$, the contract treats collateral prices as 0 and bad debt as maximal. This instantly trips the settlement thresholds. The system is structurally fail-safe; if the oracle dies, the insurance automatically pays out, perfectly aligning operator incentives against coordinated liveness attacks.

## 3. Multi-Track Settlement Triggers

Lending pool insolvency is a complex event driven by two independent axes: Endogenous Supply Pressure (Utilization) and Oracle-Dependent Collateral Adequacy (Health Factor). To prevent manipulation on a single vector, the Global Settlement sequence mandates that at least 2 of 3 independent temporal tracks trigger simultaneously.

| Track | Condition | Axis | Filter |
| :--- | :--- | :--- | :--- |
| **A: Utilization Freeze** | $U_t \ge 0.99$ | Endogenous Supply Pressure | 7-Day TWMA |
| **B: Collateral Collapse** | Weighted Collateral $\le 0.25 \times P_0$ | Exogenous Price Oracles | 7-Day TWMA |
| **C: Bad Debt Accrual** | Protocol Bad Debt $> 0$ | Accounting / Solvency | 7-Day TWMA |

*   **Track A** acts as an un-gameable physical anchor. Sustaining $U_t \ge 0.99$ for 7 days naturally broadcasts an immense arbitrage yield; if the system is solvent, capital will flow in to capture the yield, breaking the attack. If the system is insolvent, withdrawals revert and capital remains physically trapped.

## 4. Symbiotic Flywheel and Cryptoeconomic Security

To further harden the oracle infrastructure, the protocol relies on the Symbiotic restaking network. 

Oracle operators stake yield-bearing assets (e.g., `wstETH`) to participate. In exchange for producing consistent, valid ZK State Attestations, they receive a proportion ($\beta$) of the CDS premium revenue generated by the protocol.

**The Economic Tri-Fold:**
This structure achieves immense capital efficiency. The operator's staked `wstETH` simultaneously functions as:
1.  The Oracle Security Bond (subject to slashing for liveness failures).
2.  The Underwriter Escrow (satisfying the Exogenous Collateral constraint).
3.  The base Ethereum Staking Yield layer.

By triply-utilizing the same capital, the Symbiotic integration provides massive cryptoeconomic security for the settlement layer while maintaining the high delta-neutral yield necessary to attract institutional underwriters.
